	02 스프링의 특징과 의존성 주입
	--------------------------------
	2.1 스프링 프레임워크의 간략한 역사
	
		프레임워크 : 뼈대나 근간을 이루는 코드들의 묶음
		사용이유   : 프로그램의 기본 흐름이나 구조를 정하고 이 구조에 자신의
					 코드를 추가하는 방식으로 개발
	    장점	   : 개발에 필요한 구조를 이미 코드로 만들어 놓았기 때문에 
	    			 필요한 부분을 조립하는 형태의 개발이 가능
	    			 품질 보장
	    			 개발 시간 단축
	    			 생산성 향상
	    Spring 2.5 : 어노테이션(Annotation)을 활용
	    Spring 3.0 : 별도의 설정 없이도 Java클래스만으로 설정 지원
	    Spring 4.0 : 모바일 환경과 웹환경에서 많이 사용되는 REST 방식의 컨토롤러 지원
	    Spring 5.0 : Reactor를 이용한 Reactive 스타일의 개발환경 지원
	    
	    2.1.1 스프링의 주요 특징
	    	POJO(Plain Old Java Object) 기반의 구성
	    	의존성주입(DI)을 통한 객체간의 구성
	    	AOP(Aspect Oriented Programming) 지원
	    	편리한 MVC 구조
	    	WAS의 종속정이지 않는 개발 환경
	    	
	    	* 의존성(Dependency) : 하나의 객체가 다른 객체 없이 제대로 된 역할을 할 수 없는 상태로
	    		                   다른 객체의 상태에 따라 영향을 받는 것을 의미
	     	* Application Context: 필요한 객체(Bean)들의 생성을 위한 설정(빈과 빈 사이의 의존관계) 정보
	     						   XML 설정, 어노테이션 설정, Java 설정 방식 이용
	     	* 빈(Bean)           : 스프링에서 관리되는 객체
	     	* AOP				 : 개발자는 비지니스 로직에만 집중
	     						   반복적인 코드의 제거, 재사용
	     						   반드시 처리가 필요한 기능(횡단관심사:보안,로그,트랜잭션)을 모듈로 분리
	     						   유지보수가 수월한 코드 구성
